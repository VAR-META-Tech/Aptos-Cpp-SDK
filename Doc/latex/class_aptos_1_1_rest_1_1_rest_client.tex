Encodes submission. This endpoint accepts an Encode\+Submission\+Request, which internally is a User\+Transaction\+Request\+Inner (and optionally secondary signers) encoded as JSON, validates the request format, and then returns that request encoded in BCS. The client can then use this to create a transaction signature to be used in a Submit\+Transaction\+Request, which it then passes to the /transactions POST endpoint. 

If you are using an SDK that has BCS support, such as the official Rust, Type\+Script, or Python SDKs, you may use the BCS encoding instead of this endpoint.

To sign a message using the response from this endpoint\+:
\begin{DoxyItemize}
\item Decode the hex encoded string in the response to bytes.
\item Sign the bytes to create the signature.
\item Use that as the signature field in something like Ed25519\+Signature, which you then use to build a Transaction\+Signature.
\end{DoxyItemize}
